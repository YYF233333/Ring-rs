# RFC: `show` 语义收敛与人体工学优先

## 元信息

- 状态：Accepted
- 作者：Ring-rs 开发组
- 日期：2026-03-07
- 影响范围：`docs/script_syntax_spec.md`、`assets/scripts/remake/ring/**/*.md`、`vn-runtime`、`host`

---

## 1. 背景与问题

在重制迁移过程中，为了快速表达“角色差分 + 站位调度”，曾引入 `showStyle` 与 `stageActor` 作为高层语义命令。  
但结合现有 `show` 语句能力与项目目标，出现了以下问题：

1. 语义重叠：`showStyle + stageActor` 合并后本质接近 `show`。
2. 术语分裂：同一意图有多种写法，增加团队沟通与维护成本。
3. 心智负担：内容侧需要判断“到底该用哪条命令”，违反人体工学目标。

结论：应收敛为 `show` 单入口，把复杂度收敛到引擎内部而非脚本层。

---

## 2. 目标与非目标

### 2.1 目标

- 以 `show`/`hide` 作为角色演出与调度的唯一公共入口。
- 保留内容脚本的简洁表达，降低内容侧记忆负担。
- 允许 `show` 内部存在 hidden state / side effect，用于处理复杂衔接。
- 保证复杂行为可预测、可复现、可调试，避免演化为意面逻辑。

### 2.2 非目标

- 不追求 Ren'Py transform / ATL 的逐项语法兼容。
- 不在此 RFC 一次性解决所有 renderer 细节（如所有图层混合优化）。
- 不强制本阶段重写所有历史脚本；允许渐进迁移。

---

## 3. 提案摘要

### 3.1 语义收敛

- 公共语法保留：
  - `show <img src="..."/> as 角色 at 位置 with 效果`
  - `hide 角色 with 效果`
- 已移除旧语法：
  - `showStyle`
  - `stageActor`
  - 说明：`assets/scripts/remake/ring/**/*.md` 已完成迁移，运行时解析分支已收口。

### 3.2 统一职责

`show` 对同一 `as` 槽位统一处理：

1. 首次入场
2. 差分切换（同位置）
3. 位置移动（同差分）
4. 差分+移动并发变化
5. 动效衔接与默认策略选择

---

## 4. 语义执行模型（建议）

针对同一角色槽位（`as`）执行 `show` 时，运行时按状态机决策：

1. **Absent -> Present**：创建实例并执行入场效果。
2. **Present(diff only)**：执行差分切换过渡。
3. **Present(pos only)**：执行位置插值/移动过渡。
4. **Present(diff + pos)**：执行复合策略。

默认复合策略建议：`diffThenMove`。  
可选策略（后续开放）：`parallel`、`moveThenDiff`。

---

## 5. Hidden State 与 Side Effect 政策

允许 `show` 在引擎内部维护隐藏状态，但必须满足以下硬约束：

1. **确定性**：同输入脚本 + 同初始状态 => 同输出行为。
2. **可观测**：调试模式可查看关键决策（策略选择、前后状态、过渡参数）。
3. **可文档化**：默认规则与优先级必须写进规范，不依赖“实现猜测”。
4. **可降级**：复杂策略失败时有明确 fallback（例如退化为 dissolve）。

---

## 6. 防意面约束（Anti-Spaghetti Guardrails）

为防止 `show` 成为“史山入口”，实现上需遵循：

1. **单一决策点**：复合策略决策集中在一个模块，不散落各处。
2. **表驱动优先**：策略用规则表/枚举，不用层层 if-else 硬编码。
3. **副作用隔离**：渲染状态更新与资源解析分层，避免双向耦合。
4. **可测试性**：每类状态迁移都要有最小测试样例（含回归快照）。

---

## 7. 迁移计划

### Phase A：规范冻结（已完成）

- 在 `docs/script_syntax_spec.md` 明确 `show` 单入口语义。
- 标注 `showStyle`、`stageActor` 为 deprecated（仅兼容解析，不建议新增）。

### Phase B：运行时实现（已完成）

- 建立 `show` 状态机（首次/差分/移动/复合）。
- 加入默认策略与 fallback。
- 增加调试日志与诊断字段。

### Phase C：脚本迁移（已完成）

- 对 `assets/scripts/remake/ring/**/*.md` 进行渐进迁移。
- 优先高密度演出章节：`3-5`、`3-7`、`ending`。

### Phase D：收口（已完成）

- 清理仓库中的 `showStyle` / `stageActor` 使用。
- 在稳定版本移除对应解析分支。

---

## 8. 验收标准

最小验收集（每类至少 1 个场景）：

1. 首次入场
2. 原地换差分
3. 同差分移动
4. 差分+移动同帧变化
5. 图层型差分资源切换

验收维度：

- 观感：过渡连续、无明显跳闪
- 逻辑：状态一致、可重放一致
- 可维护：规则能被文档完整解释

---

## 9. 风险与缓解

1. **风险**：`show` 过载导致实现复杂度上升  
   **缓解**：策略表 + 单一决策点 + 强测试覆盖。

2. **风险**：迁移期双语法并存造成混乱  
   **缓解**：明确 deprecate 时限，新增脚本禁止使用旧语法。

3. **风险**：图层资源路径不一致导致兼容问题  
   **缓解**：资源解析层提供统一抽象，渲染层只消费规范化结果。
